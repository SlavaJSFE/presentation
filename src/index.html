## Agile Methodology
## Agile Methodology * Agile is a software development methodology. * A methodology is a set of practices and guidelines.
### The Manifesto for Agile Software Development * Individuals and interactions over processes and tools * Working software over comprehensive documentation * Customer collaboration over contract negotiation * Responding to change over following a plan
## Waterfall?!? * Long development cycle * Not flexible * Delayed feedback
## A Cross-Functional Team * The team have all the skills * No need for external help
## Sprint * Usually 2-4 weeks long * Features are scoped and delivered
## Backlog * All future tasks * Prioritized * Managed by the team
## Points * Each task is estimated in points * The whole team need to agree on points
## Hard to Estimate? * Tasks go back to *backlog* if no clarity * Discuss in the meetings
## Daily Stand-up Meetings * What we are working on * What blockers we have * Introductions, announcements, farewells
## Retro * What worked * What didn't work * What to improve
